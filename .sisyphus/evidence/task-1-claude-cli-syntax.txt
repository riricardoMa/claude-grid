CONFIRMED SYNTAX FOR INTERACTIVE MODE WITH PROMPT
================================================================================

SYNTAX: claude "prompt text"

EXPLANATION:
- The help output shows: "Usage: claude [options] [command] [prompt]"
- The "prompt" is a positional argument (not a flag)
- The first line states: "Claude Code - starts an interactive session by default"
- This means running `claude "your prompt"` will:
  1. Start an interactive session
  2. Pre-load the prompt as the initial message
  3. Keep the session open for follow-up messages

CANDIDATES ANALYSIS:
✓ claude "prompt text"           → CORRECT (positional argument, interactive mode)
✗ claude -p "prompt text"        → WRONG (print mode, exits after response)
✗ claude --prompt "text"         → NOT AVAILABLE (no --prompt flag exists)
✗ claude --message "text"        → NOT AVAILABLE (no --message flag exists)

KEY DISTINCTION:
- `-p` or `--print` flag = non-interactive, prints response and exits
- No `-p` flag + positional prompt = interactive mode (default behavior)


================================================================================
CAVEATS AND ADDITIONAL NOTES
================================================================================

1. QUOTING: The prompt text should be quoted to handle spaces and special chars
   Example: claude "This is my prompt with spaces"

2. OPTIONAL FLAGS: Can combine with other flags while keeping interactive mode
   Example: claude --model sonnet "my prompt"
   Example: claude --agent reviewer "review this code"

3. EMPTY PROMPT: Running `claude` with no prompt starts interactive mode
   with an empty initial message (user types first message)

4. SYSTEM PROMPT: Can use --system-prompt to customize system behavior
   Example: claude --system-prompt "You are a Python expert" "write a function"

5. SESSION CONTINUATION: Can combine with --continue or --resume
   Example: claude --continue "follow up on previous work"

6. IMPORTANT FOR MULTI-REPO MODE:
   - Each spawned window should use: claude "prompt text"
   - Do NOT use -p flag (that would exit immediately)
   - The prompt will be pre-loaded and user can continue typing


================================================================================
RECOMMENDATION FOR CLAUDE-GRID MULTI-REPO MODE
================================================================================

For spawning multiple Claude Code windows with initial prompts:

COMMAND TEMPLATE:
  claude "Your initial prompt here"

EXAMPLE FOR MULTI-REPO:
  claude "You are working on the backend service. Here's the task: ..."
  claude "You are working on the frontend. Here's the task: ..."
  claude "You are working on the database. Here's the task: ..."

Each window will:
- Start in interactive mode
- Display the initial prompt
- Allow the user to continue the conversation
- Maintain session persistence
